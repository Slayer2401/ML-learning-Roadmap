Machine Learning
Zero to Hero

A complete day-by-day roadmap from Python basics to your first working ML models — no experience required.

Duration: 4 Weeks · 28 Days
Daily time: 2–3 hours (+ optional weekends)
Target: Absolute Beginner
Stack: Python 3.11+ · scikit-learn 1.8
// before you start
Prerequisites

🐍 Python Basics

  • Variables, data types (int, float, str, list, dict)
  • Loops (for, while) and conditionals (if/else)
  • Functions — define, call, return
  • Basic file I/O (reading .csv)
  • Installing packages with pip

📐 Math Concepts

  • Basic algebra — equations, variables
  • Fractions, percentages, averages
  • Coordinate geometry (X-Y axes, plotting points)
  • Intuition for what a "function" means
  • No calculus required — we'll explain on the fly

🛠️ Tools to Install

  • Python 3.11+ (python.org)
  • VS Code or JupyterLab
  • pip (bundled with Python)
  • Git (optional but recommended)
  • Google Colab (free cloud alternative)

🧠 Mindset

  • Comfort with trial-and-error
  • Ability to read error messages calmly
  • Curiosity over perfection
  • Daily consistency beats cramming
  • Search first, ask second

Day 1Python Jumpstart
Theory

What is programming? Python's role in data science. Install Python + VS Code. Understand REPL vs scripts.

Hands-On
  • Variables, print(), input()
  • Data types: int, float, str, bool
  • Arithmetic and string operations
Mini Task

Write a script that asks your name and age, then prints how many days you've lived.

Day 2Control Flow & Functions
Theory

How computers make decisions. Why functions avoid repetition. Scope and return values.

Hands-On
  • if / elif / else blocks
  • for and while loops
  • def keyword, parameters, return
Mini Task

Build a simple number-guessing game using a loop and conditionals.

Day 3Lists, Dicts & Files
Theory

Data structures: how we organise information. Real-world analogy: list = shopping list, dict = phone book.

Hands-On
  • Lists — indexing, slicing, append
  • Dictionaries — key-value access
  • Reading a .csv file with open()
Mini Task

Read a small CSV of 5 student grades, compute the average using only built-in Python.

Day 4NumPy Essentials
Theory

Why arrays beat lists for maths. Vectorised operations — doing maths on millions of values at once.

Hands-On
  • np.array, shape, dtype
  • Slicing, indexing, broadcasting
  • np.mean, std, min, max, sum
Mini Task

Generate 1000 random exam scores and compute class statistics using only NumPy.

Day 5Pandas & DataFrames
Theory

What is tabular data? Rows = observations, columns = features. The "spreadsheet for code."

Hands-On
  • pd.read_csv, head(), info(), describe()
  • Selecting columns, filtering rows
  • Handling NaN — isnull(), fillna()
Mini Task

Load the Titanic dataset and answer: What % of passengers survived? What was their average age?

Day 6Data Visualisation
Theory

Why charts beat tables. Types: histogram (distribution), bar (comparison), scatter (relationship), line (trend).

Hands-On
  • matplotlib.pyplot basics
  • seaborn: sns.histplot, boxplot, heatmap
  • Titles, labels, saving figures
Mini Task

Create a 4-chart figure for the Titanic dataset: age distribution, survival rate by class, fare boxplot, correlation heatmap.

Day 7Week 1 Project & Review
Review

Revisit any concept from Days 1–6 that felt unclear. Re-read your own code and add comments.

Mini Project: EDA

Pick any Kaggle beginner dataset. Write a Jupyter notebook answering: What does the data look like? Are there nulls? What are 3 interesting patterns?

Checkpoint

Can you load, clean, and plot a dataset from scratch without googling every step? ✓

Day 8What is Machine Learning?
Theory

ML vs traditional programming. Supervised, unsupervised, reinforcement learning. Features vs labels. Training a model = finding patterns.

Hands-On
  • scikit-learn API pattern: fit → predict
  • Load iris dataset from sklearn.datasets
  • Understand X (features) and y (labels)
Mini Task

Train your first sklearn model (KNN) on Iris. Print the 5 predictions vs true labels.

Day 9Data Preprocessing
Theory

Real data is messy. GIGO: garbage in, garbage out. Feature scaling (why a $1M salary shouldn't dominate age=25). Encoding categories.

Hands-On
  • StandardScaler, MinMaxScaler
  • LabelEncoder, OneHotEncoder
  • train_test_split
Mini Task

Take a mixed dataset (numerical + categorical), preprocess it fully, and confirm shapes before/after.

Day 10Linear Regression
Theory

Predicting a number: y = mx + b. The line of best fit. What MSE/MAE mean intuitively (average prediction error).

Hands-On
  • LinearRegression().fit(X_train, y_train)
  • mean_absolute_error, mean_squared_error, r2_score
  • Plotting actual vs predicted
Mini Task

Predict Boston housing prices (or California dataset). Report your R² and plot residuals.

Day 11Logistic Regression
Theory

Predicting a category: Yes/No, Spam/Not Spam. Sigmoid function (outputs 0-1 probability). Decision boundary concept.

Hands-On
  • LogisticRegression().fit()
  • accuracy_score, confusion_matrix
  • classification_report (precision, recall, F1)
Mini Task

Classify Titanic survivors. Evaluate with a confusion matrix. What does a false negative mean here?

Day 12Model Evaluation Deep Dive
Theory

Why accuracy alone is misleading (imbalanced classes). Precision vs recall tradeoff. ROC-AUC curve intuition.

Hands-On
  • roc_curve, auc from sklearn.metrics
  • Plotting ROC curve with matplotlib
  • Choosing threshold for your use case
Mini Task

Compare two classifiers (LogReg vs KNN) side-by-side using the same train/test split. Which wins and why?

Day 13Overfitting & Underfitting
Theory

The memorisation vs generalisation problem. Learning curve plots. How train/val error tells the story. Regularisation preview (L1/L2).

Hands-On
  • Plot learning curves with sklearn
  • Ridge and Lasso regression
  • Compare train vs test accuracy
Mini Task

Intentionally overfit a model (degree-10 polynomial), then fix it with Ridge. Plot both learning curves.

Day 14Week 2 Project & Review
Review

Review: preprocessing pipeline, regression, classification, evaluation metrics. Skim notes from Days 8–13.

Mini Project: House Prices

Day 15Decision Trees
Theory

How trees make decisions by splitting data. Gini impurity vs entropy. Why trees overfit easily (they memorise).

Hands-On
  • DecisionTreeClassifier
  • plot_tree visualization
  • max_depth, min_samples_split
Mini Task

Train a tree on Iris. Visualize it. Then cap max_depth=3 and compare accuracy. Which is better on test set?

Day 16Random Forests & Ensembles
Theory

Wisdom of crowds: many weak trees → one strong model. Bagging concept. Feature importance scores. Why forests beat single trees.

Hands-On
  • RandomForestClassifier
  • feature_importances_ attribute
  • n_estimators, max_features
Mini Task

Compare Decision Tree vs Random Forest on Titanic. Bar-plot the top 5 feature importances.

Day 17Bias-Variance Tradeoff
Theory

The fundamental tension in ML. High bias = too simple (underfits). High variance = too complex (overfits). The sweet spot. Visual intuition with darts.

Hands-On
  • Validation curves (sklearn)
  • Plotting error vs model complexity
  • Cross_val_score to estimate generalisation
Mini Task

Plot a validation curve for Random Forest (varying n_estimators). Identify the point of diminishing returns.

Day 18Cross-Validation
Theory

Why one train/test split isn't enough. k-Fold CV: rotate the test set. Stratified k-fold for imbalanced data. Mean ± std reporting.

Hands-On
  • KFold, StratifiedKFold
  • cross_val_score, cross_validate
  • Reporting mean ± standard deviation
Mini Task

Apply 5-fold CV to all 3 models (LogReg, DTree, RF) on same dataset. Build a comparison table.

Day 19Feature Engineering
Theory

Features are the most powerful lever in ML. Creating new columns from existing ones. Polynomial features. Log transforms for skewed data.

Hands-On
  • Manual feature creation with pandas
  • PolynomialFeatures from sklearn
  • np.log1p for skewed distributions
Mini Task

Add 3 engineered features to the house price dataset. Does your R² improve? By how much?

Day 20Unsupervised Learning: K-Means
Theory

No labels! Finding hidden structure. K-Means algorithm: centroids, assignment, update. The elbow method for choosing k. Real use case: customer segmentation.

Hands-On
  • KMeans(n_clusters=k).fit(X)
  • Inertia and elbow plot
  • Visualising clusters with scatter
Mini Task

Cluster the Iris dataset without using labels. Compare your clusters to the true species. How well did it do?

Day 21Week 3 Project & Review
Review

Review decision trees, ensembles, bias-variance, CV, feature engineering, clustering. Identify your weakest concept.

Day 22Hyperparameter Tuning
Theory

Parameters (learned from data) vs hyperparameters (set by you before training). Grid search vs random search. Why you tune on validation, not test.

Hands-On
  • GridSearchCV, RandomizedSearchCV
  • param_grid dictionaries
  • best_params_, best_score_
Mini Task

Tune a RandomForest on the spam dataset. Compare default vs tuned accuracy and training time.

Day 23ML Pipelines
Theory

Chaining preprocessing + model into one object. Why pipelines prevent data leakage. Production-readiness: one pipeline = one deployment unit.

Hands-On
  • Pipeline from sklearn.pipeline
  • ColumnTransformer for mixed data
  • Combining Pipeline with GridSearchCV
Mini Task

Refactor your house price predictor into a single Pipeline object. Run GridSearchCV on the whole pipeline.

Day 24Saving & Loading Models
Theory

Serialisation: persisting a trained model to disk. joblib vs pickle. Model versioning basics. What "deploying a model" actually means.

Hands-On
  • joblib.dump() and joblib.load()
  • Loading a model and calling predict()
  • Simple CLI script to use a saved model

Save your best spam classifier. Write a separate script that loads it and classifies 5 new SMS messages.

Day 25SVM & Naive Bayes
Theory

SVM: finding the maximum-margin boundary. Kernel trick for non-linear data. Naive Bayes: fast, probabilistic, great for text. When to use each.

Hands-On
  • SVC with rbf kernel
  • MultinomialNB for text
  • Comparing 5 algorithms head-to-head

Build an algorithm comparison table: LogReg, KNN, DTree, RF, SVM, NaiveBayes on the same dataset.

Day 26Dimensionality Reduction (PCA)
Theory

The curse of dimensionality. PCA: capturing maximum variance in fewer dimensions. When to use it: visualisation, noise reduction, speed.

Hands-On
  • PCA(n_components=2).fit_transform(X)

// self-checks
Weekly Mini Quizzes
Week 1 Quiz
What is the difference between a list and a dictionary?

A list stores ordered values by position; a dictionary stores values by named keys.

Why use NumPy instead of plain Python lists for numeric work?

NumPy arrays support vectorized operations and efficient storage for large numeric data.

What does `df.info()` tell you?

Column names, non-null counts, data types, and memory usage.

Which chart is best for a single numeric distribution?

A histogram or boxplot.

What is EDA trying to uncover?

Data shape, quality issues, patterns, relationships, and questions worth modeling.

Week 2 Quiz
Regression or classification: predicting house price?

Regression, because the target is a continuous number.

Why split data into train and test sets?

To estimate how well the model handles unseen data.

What does a confusion matrix show?

Counts of correct and incorrect predictions by class.

Why scale numeric features?

Some algorithms compare distances or optimize coefficients, so feature scale can dominate behavior.

What is data leakage?

When training uses information that would not be available at prediction time.

Week 3 Quiz
Why can decision trees overfit?

Deep trees can memorize training-specific splits instead of learning general patterns.

How does a random forest reduce variance?

It averages many different trees trained on bootstrapped data and random feature subsets.

What does k-fold cross-validation estimate?

Model performance across multiple train/validation splits.

Give one example of feature engineering.

Creating a family-size feature from sibling, spouse, parent, and child counts.

What is clustering used for?

Finding groups or structure when labels are not provided.

Week 4 Quiz
What does GridSearchCV do?

It tries parameter combinations with cross-validation and reports the best scoring setup.

Why use a Pipeline?

It keeps preprocessing and modeling together so training, validation, and prediction are consistent.

What should be saved with a trained model?

The full pipeline, package versions, evaluation metrics, and usage notes.

When might SVMs work well?

On smaller, well-scaled datasets where the decision boundary is not simple.

What does PCA trade away?

Some interpretability and detail in exchange for fewer dimensions.

// plain english reference
ML Glossary
Feature

An input column the model uses to make a prediction.

Label

The target value the model is trying to predict.

Model

A learned pattern that maps features to predictions.

Loss

A number that tells the model how wrong its predictions are during training.

Overfitting

When a model memorizes training data and performs poorly on new data.

Pipeline

A repeatable chain of preprocessing steps and a model.

Cross-Validation

Repeated train/validation splits used to estimate model reliability.

Baseline

A simple first model used as the score to beat.
